Analogy:
Without cache: Re-reading the entire book every time you turn a page. By page 100, you have re-read 5,050 pages total.

With cache: Bookmarking where you left off. You only read each new page once. By page 100, you have read exactly 100 pages.
How it works: In a Transformer, each token produces a Key (K) and Value (V) vector during attention. Without caching, generating token N requires recomputing K and V for all N tokens. With KV-Cache, previously computed K and V are stored in memory, so only the new token's K and V need to be computed. The trade-off: more memory usage for dramatically less computation.
